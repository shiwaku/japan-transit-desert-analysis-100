# 公共交通空白地域の全国100mメッシュ分析（統計メッシュ方式）

> 「実家に帰ると、車がないとコンビニにも行けない」
> 「親が運転をやめたら、買い物はどうする？」
> 「地元の友人はほぼ全員、車を持っていることが前提で生活している」

そんな地方の現実を、全国100mメッシュ（統計メッシュ方式）で可視化したのがこのプロジェクトです。

国土交通省の定義に基づき、鉄道駅・バス停への徒歩アクセスが困難な地域（公共交通空白地域）を全国規模で定量化します。250mメッシュ版（`09_transit-desert-analysis/`）の約6.25倍の空間分解能で、境界付近の分類をより精密に行います。

## 分析結果（令和2年国勢調査）

> **駅スナップ方式**: S12ホーム線形の重心1点を NE/NW/SE/SW 4象限スナップで道路ノードに紐付け（最大500m）。
> 1重心あたり最大4ノード採用。

| カテゴリ | 人口 | 割合 |
|---|---|---|
| 公共交通便利地域（鉄道駅 walk 1km以内） | **5,397万人** | 42.7% |
| 公共交通不便地域（バス停 walk 500m以内） | **5,438万人** | 43.0% |
| **公共交通空白地域** | **1,803万人** | **14.3%** |

**日本人の約7人に1人（1,803万人）が公共交通空白地域に住んでいる。**

65歳以上では **570万人**（高齢者全体の16.3%）が空白地域に居住。

### 250mメッシュ版との比較

|---|---|---|---|
| 公共交通便利地域 | 5,389万人（42.8%） | 5,397万人（42.7%） | +8万人 |

## 国交省公表数値との比較

| 比較項目 | 国交省 QGIS手順（2025年4月） | 本試算 |
|---|---|---|
| **公共交通空白地域 人口** | **約735万人（5.8%）** | **約1,803万人（14.3%）** |
| 距離計算方法 | 直線距離（バッファ） | **道路距離（Dijkstra）** |
| 駅データ | N02（2023年） | **S12（2024年・ホーム線形重心1点）** |
| バス停データ | P11（2022年） | P11（2022年） |
| メッシュ分解能 | 250m | **100m（統計メッシュ方式）** |
| 人口データ | 令和2年国勢調査 | 令和2年国勢調査 |

### 差異の主因：道路距離 vs 直線距離

本試算の空白地域人口（1,803万人）が国交省の公表値（735万人）の**約2.5倍**になっている主因は距離計算方法の違いです。

```
直線距離（国交省）       道路距離（本試算）

  駅                         駅
  ●──────────900m──────────○    ●                         ○
  直線900m < 1,000m              │ ←道路が迂回             │
  → 「便利地域」                  └──────1,300m──────────┘
                                 ネットワーク1,300m > 1,000m
                                 → 「空白地域」
```

道路が川・山・住宅地を迂回する場合、ネットワーク距離は直線距離の **1.2〜1.5倍** になることが多い。直線で800〜900m圏内の地域でも、実際の歩行距離は1km超となり「空白地域」に分類される。これが空白人口の乖離（約1,068万人分）の主要因です。

> ※ 国交省の数値は「国土数値情報とQGISを活用した交通空白地の抽出と人口分析」（2025年4月）より。直線距離バッファによる簡易推計値。

## 分析定義

国土交通省 国土数値情報QGISマニュアル（2025年4月）準拠。

> 出典: [国土数値情報とQGISを活用した交通空白地の抽出と人口分析](https://nlftp.mlit.go.jp/ksj/manual/QGIS_manual_01.htm)（国土交通省 政策統括官付 地理空間情報課、2025年4月）
>
> 参考: [「交通空白」解消に関する取組](https://www.mlit.go.jp/sogoseisaku/transport/sosei_transport_tk_000237.html)（国土交通省）

| カテゴリ | 定義 |
|---|---|
| **公共交通便利地域** | 最寄り鉄道駅まで徒歩1,000m以内 |
| **公共交通不便地域** | 鉄道駅1,000m超・最寄りバス停500m以内 |
| **公共交通空白地域** | 鉄道駅1,000m超 AND バス停500m超 |

距離は**道路ネットワーク上の徒歩距離**（walk 3.6 km/h）で計算する。

### 公式定義との相違点・優位点

| 項目 | 国交省公式手順 | 本分析 |
|---|---|---|
| 距離計算 | 直線距離 | **道路距離（Dijkstra）** |
| 駅データ | N02（全駅） | **S12（全駅・乗降客数情報付き）** |
| バス停データ | P11（2022年） | **P11（都道府県別最新）** |
| メッシュ分解能 | 250mまたは500m | **100m（統計メッシュ方式）** |
| 駅スナップ | — | **NE/NW/SE/SW 4象限の最近傍ノード（最大500m・東西口対応）** |

道路距離により、川・山で隔てられた「直線は近いが実際には遠い」ケースを正確に捕捉できる。S12を使用することで駅名・運営者・乗降客数などの属性情報も保持できる。

## 100mメッシュ仕様

100mメッシュは**統計メッシュ方式**（L3 1kmメッシュを緯度10分割×経度10分割）であり、JIS標準の2進分割メッシュ（L5=250m, L6=125m）とは**別体系**。

| 項目 | 値 |
|---|---|
| メッシュサイズ | 約100m × 100m（緯度方向 ≈92.5m, 経度方向 ≈113m @35°N） |
| コード桁数 | 10桁（例: `5238432913`） |
| コード形式 | `ppuuqrstxy`（pp=lat×1.5, uu=lon-100, q=L2行0-7, r=L2列0-7, s=L3行0-9, t=L3列0-9, x=100m行0-9, y=100m列0-9） |

### S12（駅別乗降客数）について

N02（鉄道路線）の代わりに S12 を使用。駅名・運営者・路線名・乗降客数の属性を保持できる。

- S12のジオメトリは線型（ホーム線形）→ **重心1点**をポイントとして使用
- 重心ポイントを NE/NW/SE/SW 4象限スナップで道路ノードに紐付け（スナップ駅ノード数: 66,269）
- 乗降客数フィルタリングなし（全駅使用）
- 全国 **9,954 ポイント**

## アルゴリズム

```
[stations.parquet]          [busstops.parquet]
 S12ホーム線形の重心1点               │
        │                           │
        ▼                           ▼
 NE/NW/SE/SW 4象限             最近傍1道路ノードに
 各最近傍ノードにスナップ        スナップ
（最大500m）
        │                           │
        └─────────┬──────────────────┘
                  ▼
        ┌─────────┴─────────┐
        ▼                   ▼
 Multi-source Dijkstra  Multi-source Dijkstra
   （駅 → 全ノード）       （バス停 → 全ノード）
        │                   │
        └─────────┬──────────┘
                  ▼
     アクセスリンク経由でメッシュ単位の時間に変換
     dist_to_station[mesh] = dist[road_node] + acc_time
     dist_to_bus[mesh]     = dist[road_node] + acc_time
                  │
                  ▼
     閾値判定（walk 3.6 km/h）
     バス停  > 8.3分（500m相当）  → far_bus
     鉄道駅  > 16.7分（1,000m相当）→ far_station
```

Multi-source Dijkstra はスーパーソースノード方式で実装。駅・バス停をそれぞれ一括投入し、1回の探索で全メッシュへの最短徒歩時間を算出する。

駅スナップにはS12ホーム線形の重心1点を使用し、重心ポイントから NE/NW/SE/SW 4象限の最近傍ノードをスナップする。`--station-max-dist`（単位: m）で象限探索の最大距離を変更可能（デフォルト500m）。

## 既知の制約

- **道路ネットワーク**: 車道中心線ベースのため、歩道専用路・公園内通路等は対象外
- **バス運行本数**: 本数が極端に少ない停留所も「バス停あり」として扱う（GTFSデータ未使用）
- **S12 乗降客数**: 2024年時点。廃線後のデータ更新ラグが生じる可能性がある
- **一方通行**: 道路ネットワークは双方向として処理（一方通行未考慮）
- **駅スナップ距離**: `--station-max-dist`（デフォルト500m）。緯度方向近似（111,000m/°）のため、経度方向は約449m相当
